1. Two Sum

  • https://leetcode.com/problems/two-sum/

  • Hash tabel

  • 入门题目。最直接的想法当然是暴力循环，但是时间复杂度比较高，不太优雅。其实这个题目与“找到list中特定元素位置”意思相同，只不过这里绕了一个小弯，要找的是list中两个元素相加等于特定元素时两个元素的位置。其实也很简单，先让 target - element[i] 得到一个特定元素，然后按hash tabel查找的思路进行查找便可。

    • 构造hash tabel。

    • 一次循环，得到对应每个element[i]的特定查找值。对于得到的v值，进行hash tabel的查找即可，因为假设只有一个solution，所以找到后返回即可。

20. Valid Parentheses

  • https://leetcode.com/problems/valid-parentheses/

  • Stack

  • 括号配对问题，入门级别。能够实现的方法应该有很多，自己做的时候也有很多想法，但是感觉都不够优雅。最后才发现，使用stack数据结构才是解这种配对问题的正确思路。使用stack的话，问题就变得很直接。把不同类型的括号一次入栈，如果入栈元素与栈顶元素相配对，则将两者消除。如此，若所有括号都能成功配对，则最后栈的长度应为0。

    • 极端条件。

    • 这里有一个小trick，可以利用dict的性质，快速优美的对括号配对情况进行判断。

    • 一次循环，将各种括号依次入栈，并且在入栈过程中对配对进行判断。这里有一点需要注意，循环里面第一个判断不能少，因为不判断i是否在dict_paired中的话，下一个判断中的dict_paired[i]会报错。

    • 最后判断stack的长度，以确定返回结果。

53.Maximum Subarray

  • https://leetcode.com/problems/maximum-subarray/

  • 动态规划

  • 很经典的动态规划题目。求解这种连续问题的时候其实是需要知道以每个点为结尾的最大值情况，之所以是要以每个点为结尾，是因为需要满足连续性。

    • 需要维护一个与输入长度等长的result列表，用来存储每个点的情况。

    • 遍历result，把以每个元素为结尾的最大值情况计算出来。具体的计算方法是：若上一个节点的最大值>0，则以当前结点为结尾的最大值等于当前值加上上一个节点的最大值，反之不加。

    • 返回result中的最大值即可。

  • 上面的解法是很经典的动态规划的解法，但实际上这样维护列表的方式比较耗时。对于这种取最大值的问题，可以只设定一个最大值，然后在遍历过程中及时更新，这样比较节省时间和空间。

70.Climbing Stairs

  • https://leetcode.com/problems/climbing-stairs/description/

  • 递归

  • 其实就是斐波那契数列的求解。直观理解，result[n] = result[n-1] + result[n-2]，因为跨到当前楼梯的途径只有两种，一种是走一步，一种是走两步。

    • 先给极端情况。

    • 再给边界条件。

    • 最后给循环规则，返回即可。

101.Symmetric Tree

  • https://leetcode.com/problems/symmetric-tree/description/

  • 递归&&树

  • 递归与树相结合，题目就开始变的灵活，此类题目如果不用递归的话，就会变得非常复杂。该题目灵活的地方在于是使用两个并列的递归进行求解，原因是这里的对称不是局部的对称，而是全局的对称。一句总结：左左要和右右同，左右要和右左同。

    • 先给极端情况。

    • 再给递归入口，与最后返回的结果。

    • 最关键是 Symmetric 函数怎么定义。

      • 首先也要给出两种极端情况，分别是left node与right node均为None，此时是对称的，要返回True.
      • 如果left node 与 right node中一个是None，另一个不是，则返回False。
      • 如果left node 与 right node的val不相等，则直接返回False，因为有一个不对称，则是不对称。
      • 最关键的一步是接下来递归要拆开成两路，一路是左左与右右，一路是左右与右左，最后取交返回.

  • 递归算法非常经典，而树的结构正是天然的递归形式，所以递归树非常关键。

136.Single Number

  • https://leetcode.com/problems/single-number/description/

  • Hash table

  • 该题目要求在线性时间内求解问题，所以一次循环+查找的思路都不太对，因为内嵌的查找算法时间复杂度也不是线性的，不满足要求。而这种时间严格要求，但是空间约束不太严格的题目，hash table是比较直观的思路。所谓hash table，就是以value作为索引，而不是用顺序index作为索引。

    • 先构造一个空的hash table，其实就是构造一个字典。

    • 一次循环，并且同时判断若已经出现在了hash table中，则标记为1，否则标记为0。这里有一个小技巧，进行判断时，可以使用python字典内嵌的result.get()函数，但是这是时间花费比下面的做法要高，建议使用下面的方法。

    • 再来一次循环，找到标志为0的元素即可。

141.Linked List Cycle

  • https://leetcode.com/problems/linked-list-cycle/description/

  • Linked list

  • 题目稍微有点儿意思，主要是提供一种新的解题思路。如果链表里面有环，那么从头出发的两个一快一慢指针必定会相遇，抓住这一点即可。

    • 列出极端情况。

    • 分别定义从头出发的两个指针。因为第一步已经判断了极端情况，所以可以将两个指针位置定义的不同。

    • 定义while循环，判断标准就是两个指针是否相遇，如果相遇了则跳出循环，返回True；没有相遇的话，则必定在某个时刻point_1会到达链表尾部，此时返回False即可。这里有个小技巧，即无需判断point_2.因为point_2总是落后point_1，无需着重考虑。

169.Majority Element

  • https://leetcode.com/problems/majority-element/description/

  • Hash table

  • 很直观的解题思路，以数组value构造hash tabel，然后一次循环，遇到不同的value，令hash table中相应值加1。题目给定大于n/2的条件，所以遍历过程中判断hash table中是否有值大于n/2，一旦出现直接返回即可.

    • 构造空的hash table。

    • 一次遍历，对于出现的元素对应hash table中的值加1。并且要注意判断是否满足返回条件。

  • 其实该题目还有一个非常简洁，巧妙的解法。因为题目要求majority element的出现次数要大于n/2，那么对nums进行排序，n/2位置处的元素当然就是majority element了。

    • 先排序，再返回。

198.House Robber

  • https://leetcode.com/problems/house-robber/description/

  • 动态规划

  • 和斐波那契数列有点儿类似的感觉，也是从前往后进行循环，最后取一个max。这里的最优子结构其实也很明显：result[i] = max(result[i-1], result[i-2]+nums[i])，直观上的意义就是对于第i家，如果进行了抢劫，则抢劫的最大值就是当前第i家的金额+抢劫至i-2家金额（直接相加是因为全是正数，肯定越多越好），如果不抢劫那么能获得的最大值就是抢劫至上一家的最大值。

    • 构造一个等长list，用来存放不同节点处的最大值。

    • 给出一些极端条件。

    • 定义好result的前两项，注意result[1]的定义情况。

    • 从前往后遍历，并每次满足最优子结构。

    • 选择最大值返回即可。

206.Reverse Linked List

  • https://leetcode.com/problems/reverse-linked-list/

  • Linked list

  • 这个题目是非常基础的链表题，只要搞清楚链表组织形式即可。唯一要注意的一点就是要先从pre=None开始，最后返回的头指针也是pre。

    • 确定初始条件。

    • 循环调整pre，curr两指针的顺序，并最终返回头指针pre。之所以返回pre，是因为pre=curr执行过之后才跳出了while循环。

234.Palindrome Linked List

  • https://leetcode.com/problems/palindrome-linked-list/

  • Linked list

  • 非常基础的题目，最直观的做法就可以解这个问题。先顺序遍历链表，把值都存在一个list里面，然后比较list正反序情况即可

    • 极端条件。

    • 顺序读取链表，并存成一个list。

    • 令value list倒序，这里python有一个简洁的方式进行倒叙。list[::-1]即取整个list的倒序，list[n::-m]则是取m至n的部分倒序。

    • 线性循环比较正序、倒序结果即可。

283.Move Zeroes

  • https://leetcode.com/problems/move-zeroes/description/

  • Sort

  • 首先这个题目不能用冒泡排序解，因为时间复杂度太高。尝试使用快排的思想进行求解，可以在线性时间复杂度内解决问题。具体来讲，就是一次遍历，出现非零值就与“前方”的元素进行交换。注意，这里“前方”的元素指的是未被交换过的“前方”元素。

    • 先定义“前方”元素，因为循环还没有开始，那么前方元素就是第一个元素。

    • 一次循环，发现非零元素就进行交换，交换过后，更新“前方”元素的索引。
