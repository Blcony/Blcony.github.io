1. Two Sum

  • https://leetcode.com/problems/two-sum/

  • Hash tabel

  • 入门题目。最直接的想法当然是暴力循环，但是时间复杂度比较高，不太优雅。其实这个题目与“找到list中特定元素位置”意思相同，只不过这里绕了一个小弯，要找的是list中两个元素相加等于特定元素时两个元素的位置。其实也很简单，先让 target - element[i] 得到一个特定元素，然后按hash tabel查找的思路进行查找便可。

    • 构造hash tabel。

    • 一次循环，得到对应每个element[i]的特定查找值。对于得到的v值，进行hash tabel的查找即可，因为假设只有一个solution，所以找到后返回即可。

13.Roman to Integer

  • https://leetcode.com/problems/roman-to-integer/

  • List

  • 思路比较简单，先把str拆开，然后找到对应的数字，然后从左到右加起来，遇到当前比下一个小的情况变成减法即可。

    • 这种拆分映射的方式还是比较简洁高效的。

    • 按照从左往右的顺序拆分相加即可。

20. Valid Parentheses

  • https://leetcode.com/problems/valid-parentheses/

  • Stack

  • 括号配对问题，入门级别。能够实现的方法应该有很多，自己做的时候也有很多想法，但是感觉都不够优雅。最后才发现，使用stack数据结构才是解这种配对问题的正确思路。使用stack的话，问题就变得很直接。把不同类型的括号一次入栈，如果入栈元素与栈顶元素相配对，则将两者消除。如此，若所有括号都能成功配对，则最后栈的长度应为0。

    • 极端条件。

    • 这里有一个小trick，可以利用dict的性质，快速优美的对括号配对情况进行判断。

    • 一次循环，将各种括号依次入栈，并且在入栈过程中对配对进行判断。这里有一点需要注意，循环里面第一个判断不能少，因为不判断i是否在dict_paired中的话，下一个判断中的dict_paired[i]会报错。

    • 最后判断stack的长度，以确定返回结果。

21.Merge Two Sorted Lists

  • https://leetcode.com/problems/merge-two-sortedp-lists/

  • List

  • 这个题目感觉还是比较直观的，因为没有要求不能占用其余的空间。所以可以新建一个list，然后依次取l1, l2的元素，进行比较后，插入新建的list即可。

    • 新建一个新的链表。这里注意不仅要记录node，而且要记录root节点。

    • 依次比较后对新建的空list进行插入，这里要注意node也要依次后移。

    • l1或者l2为空之后，直接把余下的部分接到新建list后即可。

    • 最后要注意返回的是root.next，因为新建的第一个节点为了索引并不需要返回。

53.Maximum Subarray

  • https://leetcode.com/problems/maximum-subarray/

  • 动态规划

  • 很经典的动态规划题目。求解这种连续问题的时候其实是需要知道以每个点为结尾的最大值情况，之所以是要以每个点为结尾，是因为需要满足连续性。

    • 需要维护一个与输入长度等长的result列表，用来存储每个点的情况。

    • 遍历result，把以每个元素为结尾的最大值情况计算出来。具体的计算方法是：若上一个节点的最大值>0，则以当前结点为结尾的最大值等于当前值加上上一个节点的最大值，反之不加。

    • 返回result中的最大值即可。

  • 上面的解法是很经典的动态规划的解法，但实际上这样维护列表的方式比较耗时。对于这种取最大值的问题，可以只设定一个最大值，然后在遍历过程中及时更新，这样比较节省时间和空间。

    • 先给极端情况。

    • 再给边界条件。

    • 最后给循环规则，返回即可。

101.Symmetric Tree

  • https://leetcode.com/problems/symmetric-tree/description/

  • 递归 & Tree

  • 递归与树相结合，题目就开始变的灵活，此类题目如果不用递归的话，就会变得非常复杂。该题目灵活的地方在于是使用两个并列的递归进行求解，原因是这里的对称不是局部的对称，而是全局的对称。一句总结：左左要和右右同，左右要和右左同。

    • 先给极端情况。

    • 再给递归入口，与最后返回的结果。

    • 最关键是 Symmetric 函数怎么定义。

      • 首先也要给出两种极端情况，分别是left node与right node均为None，此时是对称的，要返回True.
      • 如果left node 与 right node中一个是None，另一个不是，则返回False。
      • 如果left node 与 right node的val不相等，则直接返回False，因为有一个不对称，则是不对称。
      • 最关键的一步是接下来递归要拆开成两路，一路是左左与右右，一路是左右与右左，最后取交返回.

  • 递归算法非常经典，而树的结构正是天然的递归形式，所以递归树非常关键。

    • 首先定义递归函数，当递归到叶子节点的孩子节点时，返回0。然后按照刚刚的规则进行计算即可。

    • 主函数里面直接调用即可。

136.Single Number

  • Hash table

    • 先构造一个空的hash table，其实就是构造一个字典。

    • 再来一次循环，找到标志为0的元素即可。

141.Linked List Cycle

  • https://leetcode.com/problems/linked-list-cycle/description/

  • Linked list & 双指针法

  • 题目稍微有点儿意思，主要是提供一种新的解题思路。如果链表里面有环，那么从头出发的两个一快一慢指针必定会相遇，抓住这一点即可。

    • 列出极端情况。

    • 分别定义从头出发的两个指针。因为第一步已经判断了极端情况，所以可以将两个指针位置定义的不同。

    • 定义while循环，判断标准就是两个指针是否相遇，如果相遇了则跳出循环，返回True；没有相遇的话，则必定在某个时刻point_1会到达链表尾部，此时返回False即可。这里有个小技巧，即无需判断point_2.因为point_2总是落后point_1，无需着重考虑。

160.Intersection of Two Linked Lists

  • https://leetcode.com/problems/intersection-of-two-linked-lists/

  • Linked list & 双指针法

  • 双指针法应用到这个题目还真的挺有意思，可以看成141题目的进阶形式。其实还是可以想到对两个链表进行循环遍历，两个指针第一次相遇的地方就是两个链表合并的地方，但是这种做法有个问题，对于两个链表压根没有相遇的情况，无法处理。因为若两个链表本身没有合并，那么while循环将一直持续下去。所以巧妙的点在于，可以使两个链表交互起来，A链表循环到头之后再从B链表开始循环，B链表循环到头之后再从A链表开始循环。

  • 这种让两个链表交互的操作，既不会破环首次相遇就是合并处的性质，而且即使两个链表没有合并，两个指针也会在同时为None的时候退出while循环，不会出现死循环的情况，这就是很巧妙的一点。具体一点儿，可以理解为两个没有合并的链表合并于None，那么两个指针肯定会相遇在None处，然后退出。

169.Majority Element

  • https://leetcode.com/problems/majority-element/description/

  • Hash table

  • 很直观的解题思路，以数组value构造hash tabel，然后一次循环，遇到不同的value，令hash table中相应值加1。题目给定大于n/2的条件，所以遍历过程中判断hash table中是否有值大于n/2，一旦出现直接返回即可.

    • 构造空的hash table。

    • 一次遍历，对于出现的元素对应hash table中的值加1。并且要注意判断是否满足返回条件。

  • 其实该题目还有一个非常简洁，巧妙的解法。因为题目要求majority element的出现次数要大于n/2，那么对nums进行排序，n/2位置处的元素当然就是majority element了。

    • 先排序，再返回。

198.House Robber

  • https://leetcode.com/problems/house-robber/description/

  • 动态规划

  • 和斐波那契数列有点儿类似的感觉，也是从前往后进行循环，最后取一个max。这里的最优子结构其实也很明显：result[i] = max(result[i-1], result[i-2]+nums[i])，直观上的意义就是对于第i家，如果进行了抢劫，则抢劫的最大值就是当前第i家的金额+抢劫至i-2家金额（直接相加是因为全是正数，肯定越多越好），如果不抢劫那么能获得的最大值就是抢劫至上一家的最大值。

    • 构造一个等长list，用来存放不同节点处的最大值。

    • 给出一些极端条件。

    • 定义好result的前两项，注意result[1]的定义情况。

    • 从前往后遍历，并每次满足最优子结构。

    • 选择最大值返回即可。

283.Move Zeroes

  • https://leetcode.com/problems/move-zeroes/description/

  • Sort

  • 首先这个题目不能用冒泡排序解，因为时间复杂度太高。尝试使用快排的思想进行求解，可以在线性时间复杂度内解决问题。具体来讲，就是一次遍历，出现非零值就与“前方”的元素进行交换。注意，这里“前方”的元素指的是未被交换过的“前方”元素。

    • 先定义“前方”元素，因为循环还没有开始，那么前方元素就是第一个元素。

    • 一次循环，发现非零元素就进行交换，交换过后，更新“前方”元素的索引。

437.Path Sum III

  • https://leetcode.com/problems/path-sum-iii/description/

  • 递归式 & DFS

  • 这类题目的思路还是不太清晰，主要是对递归树的一些固定解法不够熟练，这种算法的解题思维还没有养成。那么本题其实可以使用双重DFS的思路进行求解。DFS函数有两个参数，一个是当前节点，另一个是想要得到的值。当前节点的值等于想得到的值时说明是可行节点，再求左右子节点的可行解的个数，求和即为所有。所谓双重DFS的意思是，求解以当前节点开始的解个数时需要用到DFS，而求解以当前节点左右子节点开始的解个数时也需要用到DFS。

    • 主函数定义。这里要注意的就是以当前节点开始搜索，然后以左右节点开始递归。

    • DFS函数定义。简洁的思路，只不过带了一个参数而已。

448.Find All Numbers Disappeared in an Array

  • https://leetcode.com/problems/find-all-numbers-disappeared-in-an-array/

  • Set 操作 & In-place 操作

  • 这个题目如果使用额外的空间，还是比较简单，即O(n)时间和O(n)空间。但要求O(n)时间和O(1)空间的话，操作就比较巧妙。

    • O(n)时间和O(n)空间。想法还是比较直接暴力，先构建出来groundtruth，然后利用set差集的性质，进行求解即可。

    • O(n)时间和O(1)空间。O(1)空间就必须是原址操作了，这里一个很巧妙的想法就是，利用题目中限定为正的规则，对遍历到的元素对应的位置进行取反操作，这样经过依次遍历，出现过的元素对应位置都应为负，最后找出元素值为正的部分即可。这里需要注意的一点是，因为有的元素会重复出现，所以取反时要注意是对绝对值取反，且取对应位置时也要注意取绝对值。

543.Diameter of Binary Tree

  • https://leetcode.com/problems/diameter-of-binary-tree/

  • Binary tree & Depth

  • 这个题目算是二叉树求深度的变形。Diameter可以通过该节点的左右depth计算获得，其实也比较好理解，通过该节点的diameter就是左右子节点depth相加。

    • 还有一个地方需要注意，因为最大的diameter不一定通过root节点，所以递归到每一个节点都要比较一下大小，而且计算的值和返回的值不一样，返回时用max算depth，但是计算时用相加。

581.Shortest Unsorted Continuous Subarray

  • https://leetcode.com/problems/shortest-unsorted-continuous-subarray/

  • List

  • 题目很直接，先进行排序，然后取排序前后list中最前、最后的不相同元素的index，然后返回之间的长度即可。

    • 先进行排序，这里注意排序前先深拷贝，不然会影响list本身，声明一下index1和index2。

    • 从前到后查看，发现不同，记录index，然后break。注意如果找不到，直接返回0。

    • 从后到前查看，发现不同，记录index，然后break。注意如果找不到，这里也不需要刻意返回，因为index2找不到index1肯定也找不到，已经考虑了。

    • 最后返回index2与index1之间的距离即可。

617.Merge Two Binary Trees

  • https://leetcode.com/problems/merge-two-binary-trees/

  • Binary tree

  • 关于树的操作，感觉还是要多做一些才能熟练，简单的题目都是递归的操作，递归中要注意的还是return以及叶子节点(极端条件)的情况。

    • 定义递归函数。题目讲的是两棵树的合并，其实这里不需要开拓新的内存空间，直接将root2合并到root1即可。一般情况就是对root1.val进行修改，然后递归到root1.left以及root1.right。如果到达叶子节点，则需要进行判断。如果root2是空的，那么直接返回root1即可，因为右边的树没有合并的内容。如果root1是空，那么把右边的树挪过来再返回即可。

    • 对root调用函数后返回root即可
