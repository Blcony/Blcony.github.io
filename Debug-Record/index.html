Debug Record

Posted on 2019-06-27 | In Program , Tensorflow

记录项目中有关tensorflow代码的debug经历

Autoencoder

–> 用 tf.train.slice_input_producer 读入图片时其实时二维的，并非三维，数组维度出错。

–> 这个报错其实比较泛泛，意思时graph里面定义的tensor无法被迭代调用，不像list之类的可以用for循环调用，这也是静态图的一大弊病。

–> 最后错误是执行函数后，缺少了im_recon这个返回值。

RegisNet

–> image warp之后，tensor的形状出现[?,?,?,?]。这是因为运行 num_batch, height, width, channels = tf.unstack(tf.shape(im)) 这一句后，提取出来的shape其实是tensor，并不是数。再用tensor当作tensor的形状，这显然是不对的。

–> 在获得静态图中tensor的shape并想继续使用时，真正要使用语句为num_batch, height, width, channels = im.shape.as_list()。

–> 问题出在feature_m = Encoder(im2_warped)上。当我在同一个graph里面使用autoencoder的 pre-trained model 时，没有设置autoencoder里面的变量可以重复使用，改为 feature_m = Encoder （im2_warped, reuse=True)即可

–> 语句 variables_to_restore = slim.get_variables_to_restore(include=nets_to_restore) 是确定自己定义的图里哪些变量是需要restore的。而之前定义的图里，autoencoder是定义在loss2的命名空间下，所以 nets_to_restore = [‘Autoencoder’] 找不到相应的变量，只有改成 nets_to_restore = [‘loss2/Autoencoder’] 才能找到变量，但这样的话和之前训练好 autoencoder 的变量名又不一致。

–> 只好把 losses = compute_losses(im1, im2, flow * flow_scale) 摘除到 with tf.variable_scope(layer_name): 外面。

–> 输入Autoencoder的数据应该是经过归一化的，而现在是直接送入raw image，这肯定会有所影响。

–> scale1 = tf.cast(tf.divide(im_shape, tf.constant([2, 2])), tf.int32)，要把除出来的数变为tf.int32的类型，因为tf.image.resize_bilinear需要的参数是tf.int32。

ConyNet

–> 这个问题还是很隐蔽的，而且最后也不知道本质问题在哪里。表面问题是因为初始化Trainer时，loss_fn传入的不太对，不能直接传入einput.input()，最后还是改成了关于einput的lambda 函数形式。

–> 这个问题就显得很棘手了，因为没有报错信息，直接被killed。最开始时以为code某个部分又内存泄露的问题，但最后单步调试发现原因是，定义的网络太大，定义的参数量太多，sess.run(tf.global_variables_initi alizer()) 根本无法初始化所有变量。

–> 把网络规模缩小，减小一些参数就可以了。

IRR_net

–> 这个问题当时很让我奇怪，因为同样的代码之前已经成功跑通了。后来发现原来，使用多卡train，保存model之后，想再读出model，需要调用同样的多卡函数。

–> 在加载之前训练好的model时，需要同样对model加上多卡的设置: model = nn.DataParallel(model)

–> 这个问题的原因是，pytorch中对model定义init函数时，如果需要一个列表用来存储不同conv，不能使用python自带的list数据结构，而是应该使用nn.ModuleList() 函数创建一个有相同功能的list。

–> 然后用for循环，把需要的conv append到nn.ModuleList() 中，这样问题就会得到解决。
